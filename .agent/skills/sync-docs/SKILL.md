# Sync Docs

Ensure documentation stays current with code changes.

## Key Docs

- `README.md` - Project overview, setup
- `ARCHITECTURE.md` - System design
- `CONTRIBUTING.md` - Development guide
- `docs/` - API and feature docs

## When to Sync

- After major feature additions
- When API changes
- When architecture evolves
- When setup process changes

## Checklist

### README.md

- [ ] Features list current
- [ ] Setup instructions work
- [ ] Tech stack accurate

### Code Comments

- [ ] JSDoc on exported functions
- [ ] Complex logic explained
- [ ] Type documentation current

### API Docs

- [ ] Endpoints documented
- [ ] Request/response schemas
- [ ] Error codes listed

## Update Process

1. Identify what changed in code
2. Find affected documentation
3. Update to match current state
4. Verify accuracy
